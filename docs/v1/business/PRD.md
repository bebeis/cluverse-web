# 1) PRD (Product Requirements Document)

## 1.1 문서 정보

* 문서명: Cross-University Interest Community & Group Hub PRD
* 버전: v1.0
* 작성일: 2026-02-27
* 대상: 기획/디자인/개발/운영/CS

## 1.2 배경 및 문제 정의

### 문제점

* 기존 대학 커뮤니티(예: 학교별 커뮤니티)는 **학교 단위로 소통이 단절**됨.
* 같은 학과/관심사를 가진 학생들도 **같은 학교 내부에서만** 교류가 이루어져,

  * 타 학교의 수업/진로/연구/대외활동/공모전/인턴 정보 교류가 어렵고
  * 프로젝트/스터디/동아리 등 협업 네트워크 형성이 제한됨.

### 해결 방향

* **학과별/관심사별**로 커뮤니티를 구성하여, **학교 경계를 넘어** 연결되도록 한다.
* 서비스 내에서 **소규모 그룹**(프로젝트/동아리/스터디) 생성 → **모집/지원/승인/회원관리** → **전용 게시판** 제공.

## 1.3 제품 비전 / 한 줄 정의

* 비전: “대학 간 장벽을 없애고, 관심사 기반으로 연결되는 학생 협업 네트워크”
* 한 줄 정의: **“학과·관심사로 묶이고, 그룹으로 협업하는 전국 대학생 커뮤니티”**

## 1.4 목표(Goals) / 비목표(Non-goals)

### Goals

1. **학과/관심사 기반** 커뮤니티를 통해 타 학교 학생과의 소통량을 증가
2. **그룹 모집 → 운영**(전용 게시판/회원관리)까지 한 흐름으로 제공
3. 학생 인증 기반으로 **신뢰/안전**을 확보(익명성을 유지하되 악용 방지)
4. 초기 네트워크 효과를 위해 **발견/추천/검색** 강화

### Non-goals (v1 범위 밖)

* 실시간 음성/영상 통화
* 과제 대행/불법 거래/중고 장터(운영 리스크 큼)
* 학사행정 서비스(수강신청, 학적 등) 연동
* 전면 유료화(초기에는 성장 우선)

## 1.5 타겟 사용자 / 페르소나

1. **전공 네트워킹형**: “같은 학과 타 학교 학생들과 정보/자료/진로를 공유”
2. **프로젝트 리더형**: “팀원을 모집하고 운영(공지/자료/회의)을 한 곳에서”
3. **신입/전과/복수전공**: “커리큘럼/수업 난이도/진로를 빠르게 파악”
4. **대외활동/공모전형**: “관심사가 맞는 팀원/파트너를 찾고 싶음”

## 1.6 핵심 가치 제안(Value Proposition)

* 학교가 아니라 **관심사/학과가 중심**인 커뮤니티 구조
* 학생 인증 기반으로 **퀄리티 높은 사용자 풀** 확보
* 그룹 모집/운영이 **게시글 하나로 끝나지 않고**, “지원-승인-관리-게시판”까지 이어짐

## 1.7 사용자 시나리오(User Journey)

### 시나리오 A: 학과 커뮤니티 참여

1. 가입/인증(학교 이메일 또는 대체 인증)
2. 전공/학과 선택 + 관심 태그 선택
3. “전국 컴퓨터공학” 보드에서 글 읽기/검색
4. 질문 글 작성(익명/반익명) → 댓글/답변 수신 → 북마크

### 시나리오 B: 프로젝트 팀원 모집 → 그룹 운영

1. “AI 프로젝트” 그룹 생성
2. 모집 공고 작성(필요 역할/기간/온라인 여부/기술스택)
3. 지원자 접수(프로필 요약/포트폴리오 링크/질문)
4. 리더가 승인/거절
5. 그룹 전용 게시판(공지/회의록/자료/일정 링크)에서 운영

## 1.8 범위(Scope) – MVP / Post-MVP

### MVP (v1)

* 학생 인증/가입
* 프로필(학교/학과/관심 태그)
* 학과/관심사 보드(피드/글/댓글/반응/북마크)
* 검색/추천(기본)
* 그룹(생성/모집/지원/승인/멤버관리)
* 그룹 전용 게시판(공지/일반)
* 신고/차단/모더레이션 기본 도구
* 알림(댓글/답글/승인/공지)

### Post-MVP (v1.5~)

* 그룹 캘린더/일정 관리
* 그룹 파일 보관/링크 아카이브
* 실시간 채팅(그룹 채팅)
* 고급 추천(개인화), 학교/학과 인증 배지 확장
* 멘토링/선후배 매칭(정책/리스크 검토 후)

## 1.9 성공 지표(Metrics)

* Activation
  * 가입 후 24시간 내 “팔로우/태그 선택 + 1회 이상 피드 상호작용” 비율
* Engagement
  * WAU/MAU, 글 작성률, 댓글 작성률, 북마크/좋아요 비율
  * **교차 학교 상호작용률**(댓글/답글의 타학교 비중)
* Community Health
  * 신고율(DAU 대비), 신고 처리 SLA, 제재 후 재발률
* Groups
  * 그룹 생성 수, 모집 공고 전환율(지원/승인), 그룹 2주 유지율(활동 게시글 존재)

## 1.10 리스크 및 대응

* 인증 허들로 인한 이탈 → 대체 인증 수단/단계적 인증(읽기 전용 → 작성 권한)
* 악성 사용자/분쟁 증가 → 신고 카테고리/제재 매트릭스/운영툴 강화
* 네트워크 효과 부족 → 초기에는 특정 학과/관심사 “씨앗 커뮤니티” 전략 필요
* 개인정보/법적 이슈 → 최소 수집/암호화/보관기간/삭제요청 프로세스 명문화

# 2) 기능 정의서 (Functional Specification)

## 2.1 기능 목록(모듈)

1. 회원/인증
2. 프로필/관심사 설정
3. 보드(학과/관심사) & 피드
4. 게시글/댓글/반응
5. 검색/발견(탐색)
6. 알림
7. 그룹(프로젝트/동아리/스터디)
8. 그룹 모집/지원/승인/멤버관리
9. 그룹 전용 게시판
10. 신고/차단/모더레이션
11. 관리자 콘솔(운영툴)
12. 로그/통계(내부)

---

## 2.2 회원/인증

### 목적

* “대학생 커뮤니티”의 신뢰 확보, 스팸/어뷰징 방지

### 주요 기능

* 가입: 이메일/소셜(선택) + 약관 동의
* 학생 인증(택1 또는 복수 제공)
  * 학교 이메일 인증(도메인 allowlist)
  * 대체 인증(학생증/재학증명서 업로드 → 운영 검수) ※ MVP에서는 “기능만 열어두고 수동 검수”
* 권한 단계
  * 미인증: 읽기/검색만 제한적으로 허용
  * 인증: 글/댓글/그룹 기능 전체 사용

---

## 2.3 프로필/관심사

### 필드(예시)

* 학교(필수), 학과(필수), 학번(선택), 관심 태그
  * 학과의 경우 주전공/복수전공/부전공 선택 가능해야 함
  * 관심 태그는 학과와 연관관계가 있도록 구조화
  * 유사 학과(ex. 소프트웨어공학과, 컴퓨터공학과, ...)를 잘 처리할 수 있어야 함 
* 자기소개(선택), 링크(선택: GitHub/Notion/Portfolio/Instagram/...)
* 익명 설정:
  * 프로필을 공개/비공개 선택할 수 있음 
  * 또한, 공개할 필드를 설정할 수 있음
    * 단, 프로필에서 닉네임은 기본 공개 
  * 학교의 경우 학교 배지로 보여줌 

### 동작
* 가입 직후 온보딩에서 학과/관심사 선택
* 관심 태그 기반으로 홈 피드 구성
* 특정 유저를 팔로우 할 수 있음
  * 팔로우한 유저의 활동에 대한 알림을 받거나, 팔로우한 유저들의 활동만 모아볼 수 있는 기능 제공
* 특정 유저를 차단할 수 있음

---

## 2.4 보드(학과/관심사) & 피드

### 보드 종류

* 학과 보드(예: 컴퓨터공학, 경영학 등)
    * 유사학과의 경우 하나의 공통 보드 및에 세부 보드가 구성되어야 함 
    * UI/UX를 고려하여 최대 3depth로 제한 
* 관심사 보드(예: AI, 스타트업, UX, 공모전, 교환학생, 공연, 체육, 주식, ...)

### 피드 종류(MVP)

* 홈 피드: 구독한 학과/관심사 보드 글 + 추천글/인기글 혼합
* 보드 피드: 해당 보드 내 글 목록
* 트렌딩: 최근 반응/댓글 기반 랭킹(간단 로직)

---

## 2.5 게시글/댓글/반응

### 게시글

* 작성: 제목, 본문, 태그, 카테고리, 익명 여부, 이미지 첨부
* 편집/삭제: 작성자만(단, 운영 정책에 따라 제재 가능)
* 상호작용: 좋아요(추천), 북마크, 공유(링크), 신고
* 질문/정보/후기/자료/모집... 등을 선택할 수 있음
* 조회수 / 좋아요 수 / 댓글 수 / 북마크 수를 보여줘야 함

### 댓글/대댓글

* 댓글/대댓글 작성(무한 depth), 좋아요, 신고, 삭제
    * 좋아요 수 / 대댓글의 수 / ... 이런 거 보여줘야 함 
* 차단한 사용자의 콘텐츠 숨김

### 정렬/필터

* 최신순/조회순/댓글많은순
* 태그/카테고리/... 필터

---

## 2.6 검색/발견(탐색)
API_V1에서 제외. 와이어프레임에선 포함시키고 준비중인 기능으로 처리

* 통합 검색: 글 제목/본문/태그/그룹/모집글
* 자동완성: 태그/학과/관심사
* 추천: 관심태그 기반 “추천 보드/추천 그룹”

---

## 2.7 알림

* 인앱 알림 + 푸시(옵션)
* 트리거(MVP)
  * 내 글 댓글/답글
  * 내 댓글 답글
  * 내가 지원한 그룹 결과(승인/거절)
  * 내가 운영하는 그룹에 지원 도착
  * 그룹 공지 등록

---

## 2.8 그룹(프로젝트/동아리/스터디)

### 그룹 엔터티
* 그룹명, 소개, 대표 이미지(선택), 관심태그, 활동 방식(온라인/오프라인), 지역(선택), 카테고리(ex. 동아리, 소규모, 단기 프로젝트, ...)
* 공개 범위
  * 공개(검색/탐색 노출)
  * 일부공개(검색 노출, 내부 게시판은 멤버만)
  * 비공개(초대 링크로만)

### 역할(Role)

* 오너(생성자)
* 관리자(모더레이터)
* 멤버
* 대기(지원 후 미처리)

---

## 2.9 그룹 모집/지원/승인/멤버관리

### 모집글

* 모집 포지션(예: 프론트/백/디자인), 요구 조건, 기간, 목표, 진행 방식, 모집 마감일
* 지원 방식

  * 인앱 지원서(질문 3~5개)
  * 링크 첨부(포트폴리오)

### 승인 흐름

1. 지원 제출 → 그룹 관리자 알림
2. 제출됨 -> 검토중 -> 승인/거절
- 지원자에게 채팅 가능
3. 승인 시 멤버로 편입, 그룹 게시판 접근 권한 부여

### 멤버관리

* 권한 변경(관리자 임명, 마스터 변경)
* 그룹 내 자율 직책 생성 및 부여(ex. 학술부장, 총무부장, ...)
* 강퇴/차단

---

## 2.10 그룹 전용 게시판

* 게시판 타입(MVP)
  * 공지(관리자만 작성)
  * 일반(멤버 작성)
* 기능
  * 글/댓글/핀(공지 고정)
  * 멤버 전용 열람 권한
  * 외부 공개 유무 선택 가능(단, 그룹 공개 수준 초과 불가능)
* Post-MVP: 일정/파일/투표

---

## 2.11 신고/차단/모더레이션

* 신고 대상: 글/댓글/사용자/그룹/모집글
* 신고 사유(정책과 동일한 카테고리)
* 처리 상태: 접수 → 검토 → 조치/반려 → 완료
* 차단: 상호 콘텐츠 숨김, 알림 차단

---

## 2.12 관리자 콘솔(운영툴)

* 유저 관리: 제재, 인증 상태, 활동 로그
* 콘텐츠 관리: 신고 큐, 삭제/블라인드, 복구
* 그룹 관리: 그룹 강제 비공개/폐쇄, 오너 확인
* 통계: DAU/WAU/MAU, 신고율, 그룹 전환율
